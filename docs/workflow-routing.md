# Workflow Routing

这个整合 Agent 的第一步永远是路由，而不是直接分析。

## 路由表

| 用户输入状态 | 推荐路径 | 原因 |
| --- | --- | --- |
| 只有品类想法、模糊问题、老板追问 | PM Triage | 先判断是否值得深挖，避免过早重投入 |
| 有 BSR / 主关键词 / ABA / CPC | QYPM 001-003 | 可以做公司匹配、趋势需求、市场结构 |
| 有竞品 ASIN / 前台搜索结果 | QYPM 004 | 可以做竞品初选、关键词竞争、功能拆解 |
| 有 Review / Q&A / Keepa / Listing | QYPM 005 | 可以做 VOC、FACT、JTBD、产品定义假设 |
| 有报价 / BOM / FBA / 样品 / 专利 / 合规 | QYPM 006-007 | 可以做 ROI、供应链、实物验证和风险登记 |
| 要老板汇报 / Excel / PPT | Delivery + QA | 从分析结果转为可上会交付物 |

## 资料不足时

不要进入最终产品定义、ROI 或量产判断。输出：

- 当前只能做什么
- 缺什么 P0 数据
- 补数据后能进入哪个阶段
- 下一步最小验证动作

## 资料充足时

直接进入对应 QYPM 漏斗，但必须先声明：

- 当前阶段
- 允许输出
- 禁止输出
- 数据边界
- 本轮交付物
